# SLG 自动化助手 - 产品规格说明书

> 版本：v1.0
> 更新日期：2026-05-12
> 适用产品：万国觉醒（Rise of Kingdoms）自动化辅助工具

---

## 1. 产品概述

### 1.1 产品定位
面向 SLG（策略类）手游玩家的桌面自动化辅助工具，首期支持《万国觉醒》。通过 ADB 协议控制 Android 模拟器，实现资源收集、建筑升级、城外采集、科技研究等日常运营任务的全自动化，解放玩家双手。

### 1.2 目标用户
- **核心用户：** 中重度 SLG 玩家，每日需要花费 1-2 小时处理重复操作
- **付费意愿：** 愿意为节省时间付费的成年玩家（25-45 岁）
- **使用场景：** 上班通勤期间、夜间挂机、工作时后台运行

### 1.3 商业模式
- **激活码授权制**：1 个月时限激活码
- **设备绑定**：1 个激活码绑定 1 台电脑
- **续费机制**：到期前 7 天提醒
- **定价参考**：¥30-50/月（待市场验证）

## 2. 核心功能

### 2.1 已实现功能（v0.x）

#### 2.1.1 城内资源收集
- **功能描述：** 自动遍历配置的资源建筑（农场/伐木场/矿场），点击收集资源
- **配置参数：** 资源建筑列表、收集偏移量
- **执行频率：** 用户可设置间隔（默认 5 分钟）

#### 2.1.2 建筑升级
- **功能描述：** 按队列顺序自动升级指定建筑
- **配置参数：** 升级队列（最多 5 个槽位）
- **智能判断：**
  - 升级成功 → 自动从队列移除
  - 队列工人忙 → 标记重试，不阻塞后续
  - 弹窗未找到 → 跳过当前建筑
- **图像识别：** 升级按钮、忙提示弹窗

#### 2.1.3 城外资源采集
- **功能描述：** 派出指定队伍采集指定类型/等级的城外资源
- **支持资源：** 农田、伐木场、石矿、金矿
- **支持等级：** 1-6 级
- **队伍管理：** 5 个队伍独立配置，按序号绑定
- **执行流程：** 9 步完整流程（切换城外 → 搜索 → 选类型 → 调等级 → 派出 → 选队伍 → 行军）
- **智能判断：**
  - 队伍可用性检测（像素对比按钮选中状态）
  - 不可用自动跳过下一个
  - 完成后自动回城

#### 2.1.4 科技研究
- **功能描述：** 自动滑动寻找指定科技并开始研究
- **支持科技：** 耕犁、锯木厂、铸币、机械
- **图像识别：** 科技图标模板匹配
- **失败处理：** 滑动 5 页未找到自动返回

#### 2.1.5 位置感知
- **功能描述：** 通过切换按钮像素特征判断当前在城内/城外
- **作用：** 各 action 执行前智能判断是否需要切换，避免误操作
- **API：** `getCurrentLocation()` / `ensureInCity()` / `ensureInWorld()`

### 2.2 待实现功能（v1.x 规划）

#### 2.2.1 必做功能（Phase 1 完善）
- **截图标注配置坐标**：浏览器内可视化点击建筑标记位置（替代手动输入）
- **建筑模板自动识别**：截图后自动识别常见建筑预填位置
- **坐标导入导出**：用户间分享配置
- **运行日志持久化**：日志按天分割，最近 7 天可查

#### 2.2.2 多账号支持（Phase 2）
- **多模拟器管理**：支持同时控制 2-5 个模拟器
- **账号独立配置**：每个账号独立的建筑坐标、任务队列
- **任务调度器**：账号间轮流执行，避免并发卡顿
- **资源监控面板**：所有账号资源/队伍状态总览

#### 2.2.3 桌面应用化（Phase 3）
- **Electron 打包**：单 .exe 安装即用
- **内嵌 ADB**：用户无需单独配置
- **系统托盘**：最小化后台运行，关闭窗口不退出
- **开机自启**：可选项
- **自动更新**：electron-updater 静默更新

#### 2.2.4 授权系统（Phase 4）
- **激活码生成与验证**：服务端管理
- **设备指纹绑定**：CPU + 主板 + MAC 生成唯一指纹
- **心跳检测**：每小时验证授权有效性
- **离线宽限期**：断网 24 小时内可继续使用
- **到期处理**：到期立即停用功能，保留配置数据

#### 2.2.5 高级功能（Phase 5+）
- **状态驱动调度**：截图识别游戏状态，按需触发 action
- **自动开盾**：定时开启护盾保护
- **自动派出侦察**：派遣侦察兵收集情报
- **联盟资源协助**：自动点击援助按钮
- **VIP 商店每日免费购买**

## 5. 非功能性需求

### 5.1 性能要求
- **资源占用：** 单账号 CPU < 5%，内存 < 200MB
- **图像识别耗时：** 单次模板匹配 < 500ms
- **24 小时挂机稳定性：** 无内存泄漏，无 ADB 断连
- **多账号并发：** 5 个账号同时运行总占用 < 1GB 内存

### 5.2 兼容性
- **操作系统：** Windows 10+（首期），macOS（后期）
- **模拟器：** MuMu、雷电、蓝叠、夜神
- **分辨率：** 1280x720、1600x900、1920x1080（首期）
- **游戏版本：** 万国觉醒中文版国服

### 5.3 安全性
- **授权数据加密存储**：本地激活码用 AES-256 加密
- **设备指纹防伪造**：多源信息组合，防虚拟机绕过
- **关键算法保护**：核心识别逻辑放服务端
- **代码混淆**：javascript-obfuscator 处理打包后代码

### 5.4 可维护性
- **插件化架构**：游戏更新只需改对应插件
- **图像模板独立**：模板与代码分离，热替换
- **配置导入导出**：用户配置 JSON 格式，便于支持
- **日志可追溯**：每次操作留痕，便于排查问题

## 6. 风险与缓解

### 6.1 技术风险

|------|------|---------|

## 9. 附录

### 9.1 术语表
- **ADB**：Android Debug Bridge，安卓调试协议
- **Action**：插件中的一个具体操作（如收集资源）
- **模板匹配**：通过对比图像找到屏幕中的目标元素
- **像素对比**：对比两张图的像素差异率
- **Plugin**：插件，针对特定游戏的实现集合

### 9.3 关键 API 参考

**PluginContext 提供的 API：**
- `tap(x, y)` - 点击坐标
- `swipe(x1, y1, x2, y2, duration)` - 滑动
- `sleep(seconds)` - 等待
- `findImage(template, threshold)` - 查找模板
- `findImageWithLocation(template, threshold)` - 查找模板并返回位置
- `tapImage(template, threshold)` - 查找并点击模板
- `captureRegion(x, y, w, h)` - 截取屏幕区域
- `compareImages(img1, img2)` - 对比两张图差异率
- `checkButtonStateChange(...)` - 检测按钮点击后状态变化
- `detectState(...)` - 多模板对比，返回最匹配的状态
- `getConfig(key, default)` - 读取配置
- `log(message)` - 输出日志
