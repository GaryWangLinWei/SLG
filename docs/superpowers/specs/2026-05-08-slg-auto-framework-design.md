# SLG自动化通用框架设计文档

## 1. 项目概述

### 1.1 项目背景
面向SLG（策略模拟游戏）玩家的通用自动化辅助工具，采用图像识别+模拟操作技术路线，不依赖特定游戏API，通过插件化架构实现可扩展性。

### 1.2 核心定位
- **通用框架**：不针对特定游戏，提供基础能力
- **混合模式**：开箱即用的常用功能 + 脚本扩展能力
- **面向Android模拟器**：通过ADB与模拟器通信

## 2. 需求分析

### 2.1 核心功能优先级
1. **常用场景模板**（最高优先级）：自动收资源、升级建筑等预设功能
2. **基础操作原语**：图像识别、点击、滑动、等待等基础API
3. **图像模板管理**：界面截图、模板管理、OCR文字识别
4. **任务编排系统**：条件执行、分支判断、错误处理
5. **脚本编辑器**：内置编辑器、语法高亮、调试功能

### 2.2 目标用户
- 普通玩家：使用预置插件和模板，无需编程
- 高级玩家：编写自定义脚本和插件
- 社区贡献者：分享插件到插件市场

### 3.1 架构原则
1. **核心层极简稳定** - 只做最基础的设备操作和图像识别，不包含具体游戏逻辑
2. **插件承载业务** - 所有游戏相关逻辑都在插件中实现
3. **接口标准化** - 插件与核心之间通过明确定义的API交互
4. **Web层无状态** - 通过REST API与核心引擎通信

## 4. 核心模块设计

### 4.1 设备管理器
- **职责**：管理Android模拟器连接，封装ADB操作
- **核心功能**：
  - 自动发现本地运行的模拟器/设备
  - 截图获取（adb screencap）
  - 输入事件：点击、滑动、长按、输入文本
  - Shell命令执行
- **关键抽象**：`Device` 接口，未来可扩展支持iOS/其他设备

### 4.2 图像识别模块
- **职责**：提供统一的图像识别API给插件使用
- **核心功能**：
  - **模板匹配**：在截图中查找指定图片，返回坐标和置信度
  - **OCR识别**：指定区域文字识别（支持中文）
  - **颜色检测**：检测指定位置/区域的颜色值
  - **特征点匹配**：应对缩放、旋转等变形情况
- **技术选型**：OpenCV + PaddleOCR（中文识别效果好）

### 4.3 任务调度器
- **职责**：管理自动化任务的生命周期
- **核心功能**：
  - 任务队列管理（排队、暂停、停止）
  - 定时任务（CRON表达式）
  - 条件触发（当画面出现XX时执行YY）
  - 任务状态持久化和恢复
  - 执行日志记录

### 4.4 插件系统
- **职责**：插件的加载、管理、生命周期管理
- **插件结构**：
```javascript
{
  id: "com.example.slg-common",
  name: "SLG通用插件",
  version: "1.0.0",
  author: "...",
  actions: [
    { id: "collect-resources", name: "收集资源", run: async (ctx) => { ... } }
  ],
  config: { ... },
  onLoad: async () => { ... },
  onUnload: async () => { ... }
}
```

## 5. SLG通用插件设计

### 5.1 预设操作模板

| 操作名称 | 功能描述 | 可配置项 |
|---------|---------|---------|
| **收集资源** | 遍历各个资源建筑点击收集 | 资源按钮位置、等待时间、重试次数 |
| **升级建筑** | 按优先级自动升级建筑队列 | 建筑优先级列表、升级条件判断 |
| **自动出征** | 自动选择目标、派兵、出征 | 出征目标坐标、兵种配置、出征间隔 |
| **招募士兵** | 自动在兵营训练士兵 | 兵种优先级、训练数量阈值 |
| **任务领奖** | 自动领取每日任务、活动奖励 | 任务按钮位置、奖励识别 |
| **签到打卡** | 每日自动签到 | 签到界面位置、签到按钮 |
| **扫描检测** | 扫描特定按钮/弹窗并处理 | 要检测的图像模板、触发的操作 |

### 5.2 插件上下文API

插件运行时可用的API：
```javascript
ctx = {
  // 图像识别
  findImage(templatePath, threshold=0.8)  // 查找图片
  readText(region)                         // OCR识别
  waitForImage(template, timeout=30)       // 等待图片出现
  
  // 设备操作
  tap(x, y)                                // 点击坐标
  tapImage(template)                       // 点击找到的图片
  swipe(x1, y1, x2, y2, duration=500)      // 滑动
  inputText(text)                          // 输入文本
  sleep(seconds)                           // 等待
  
  // 流程控制
  loop(count, action)                      // 循环执行
  retry(action, times=3)                   // 重试
  log(message)                             // 输出日志
  
  // 配置访问
  config.get(key)                          // 获取插件配置
}
```

## 6. 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **后端核心** | Node.js + TypeScript | 异步IO适合自动化场景，生态丰富，插件易编写 |
| **图像识别** | OpenCV.js + PaddleOCR | OpenCV做模板匹配，PaddleOCR做中文识别 |
| **Web前端** | React + TypeScript + Tailwind | 现代化UI，开发效率高 |
| **设备通信** | ADB (Android Debug Bridge) | 与模拟器通信的标准方式 |
| **数据持久化** | SQLite | 轻量级，无需独立服务 |
